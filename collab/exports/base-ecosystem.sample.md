# base-ecosystem — sample export

## What it covers

A grounded summary of Base-native building patterns, standards, and common pitfalls.

## When to use it

Use this skill when Blue Agent needs to reason about Base-specific architecture, ecosystem conventions, or launch assumptions.

## Core concepts

- Base is the default execution environment.
- Verified addresses matter.
- Launch flows should be simple, grounded, and reproducible.
- Ecosystem claims should be explicit and sourced.

## Patterns

- Prefer Base-only flows unless there is a clear reason not to.
- Keep token, wallet, and payment references canonical.
- Treat ecosystem compatibility as a first-class constraint.

## Pitfalls

- Guessing addresses.
- Mixing unrelated chains into the default path.
- Overclaiming partnerships or traction.

- [ ] No invented stats
